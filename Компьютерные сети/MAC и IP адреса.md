# MAC и IP адреса

## Общие понятия

Оба типа адресов нужны для идентификации устройств в сети - чтобы было понятно, кому отправлять данные и откуда они пришли. Но используются они по-разному.

## MAC-адрес (физический адрес)

### Определение
MAC-адрес - это уникальный идентификатор сетевой карты (сетевого адаптера, сетевухи, NIC).

### Назначение
- Идентификация устройства в локальной сети
- Используется для коммутации на канальном уровне модели OSI

### Структура
- Состоит из 12 символов
- Записывается в виде 6 пар 16-ричных чисел
- Разделители: двоеточия или дефисы
- **Первые 3 пары** - идентификатор производителя (OUI - Organizationally Unique Identifier)
- **Последние 3 пары** - уникальный номер конкретного устройства

### Пример
```
AA:BB:CC:DD:EE:FF
```

### Особенности
- Присваивается производителем на заводе
- Должен быть уникален для каждого устройства в мире
- Не меняется (хотя технически можно изменить программно)
- Используется для коммутации в локальной сети

## IP-адрес (логический адрес)

### Определение
IP-адрес - это логический адрес для идентификации устройства в глобальной сети (например, в интернете).

### Назначение
- Идентификация устройства в глобальной сети
- Используется для маршрутизации на сетевом уровне модели OSI

### Типы IP-адресов

#### IPv4
- 4 числа от 0 до 255
- Разделены точками
- Пример: `192.168.1.1`

#### IPv6
- 8 групп 16-ричных чисел
- Разделены двоеточиями
- Создан из-за нехватки IPv4 адресов
- Пример: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`

### Получение IP-адреса
- **DHCP** - автоматическое назначение
- **Статический IP** - ручное назначение (не меняется)

## Коммутация vs Маршрутизация

### Коммутация (MAC-адреса)
- Передача данных между устройствами в **локальной сети**
- Использует коммутаторы (свитчи)
- Свитч создает таблицу коммутации: порт ↔ MAC-адрес
- Анализирует MAC-адреса для определения порта назначения

### Маршрутизация (IP-адреса)
- Передача данных между устройствами в **разных сетях**
- Использует маршрутизаторы (роутеры)
- Роутер использует таблицу маршрутизации
- Анализирует IP-адреса для определения пути доставки

## ARP (Address Resolution Protocol)

### Назначение
Протокол, который связывает IP-адреса с MAC-адресами.

### Принцип работы
- Зная IP-адрес, можно узнать соответствующий MAC-адрес
- Необходим для доставки данных в локальной сети

## Практический пример

### Отправка файла на принтер (локальная сеть)
1. Компьютер использует **MAC-адрес** принтера
2. Свитч направляет данные на нужный порт по MAC-адресу

### Заход на сайт (глобальная сеть)
1. Компьютер использует **IP-адрес** сайта
2. Роутер направляет данные по IP-адресу через интернет
3. В локальной сети все равно используется MAC-адрес

## Аналогия с почтой

- **IP-адрес** = адрес доставки (помогает найти дом в городе)
- **MAC-адрес** = имя получателя (нужно для вручения конкретному человеку)

## Ключевые различия

| Характеристика | MAC-адрес | IP-адрес |
|---|---|---|
| **Тип** | Физический | Логический |
| **Область применения** | Локальная сеть | Глобальная сеть |
| **Уровень OSI** | Канальный | Сетевой |
| **Изменяемость** | Не меняется | Может меняться |
| **Назначение** | Коммутация | Маршрутизация |
| **Уникальность** | Глобально уникален | Уникален в сети |
