# Уровень представления (Presentation Layer)

## Описание
Уровень представления (Presentation Layer) - шестой уровень модели OSI, обеспечивающий преобразование данных между различными форматами представления, кодирование, сжатие и шифрование данных.

## Основные функции
- **Преобразование форматов** - конвертация данных между различными форматами представления
- **Кодирование/декодирование** - преобразование данных в стандартные форматы для передачи
- **Сжатие данных** - уменьшение объема данных для оптимизации передачи
- **Шифрование/расшифровка** - обеспечение безопасности передаваемых данных
- **Синтаксическая совместимость** - обеспечение совместимости между различными системами

## Примеры форматов данных
- **JPEG** - формат изображений
- **GIF** - формат анимированных изображений
- **MP3** - формат аудио
- **PDF** - формат документов

## Процесс работы
1. **Отправка**: Данные преобразуются в нужный формат
2. **Передача**: Данные передаются в виде нулей и единиц
3. **Получение**: Данные преобразуются обратно в читаемый формат

## Пример использования
Когда вы отправляете фотографию кота:
- Фото преобразуется в бинарный формат
- Передается по сети как последовательность 0 и 1
- На уровне получателя преобразуется обратно в изображение
- Открывается в программе просмотра

## Характеристики
- **Формат-независимость** - обеспечивает совместимость между различными форматами данных
- **Прозрачность для приложений** - приложения не знают о преобразованиях данных
- **Стандартизация** - использует стандартные форматы для межсистемного обмена
- **Безопасность** - обеспечивает шифрование и защиту данных
- **Оптимизация** - сжатие данных для эффективной передачи

## Конспект из лекции

> "Все выше и выше. На шестом уровне творится преобразование форматов сообщений, такое как кодирование или сжатие. Тут живут JPEG и GIF, например."

> "То есть, когда ты кому-то отправляешь очередную фотку своего кота, то она сначала приходит к получателю в виде нулей и единиц, а на этом уровне она преобразуется в нужный формат, который можно будет открыть в какой-то программе."
