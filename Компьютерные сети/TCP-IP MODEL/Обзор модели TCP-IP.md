# Обзор модели TCP/IP

## Что такое модель TCP/IP?
Модель TCP/IP (Transmission Control Protocol/Internet Protocol) - это четырехуровневая сетевая модель, разработанная Министерством обороны США (DoD) для обеспечения надежной связи в военных сетях. Модель стала основой современного интернета и является практической реализацией сетевого взаимодействия.

## Зачем нужна модель TCP/IP?
- **Практичность** - реальная модель, используемая в интернете
- **Простота** - меньше уровней по сравнению с OSI
- **Совместимость** - основа для всех современных сетевых технологий
- **Масштабируемость** - поддерживает сети любого размера
- **Стандартизация** - единые протоколы для глобальной сети

## Структура модели

### 1. Канальный уровень (Network Access Layer)
- Передача данных между соседними узлами
- MAC-адреса, кадры, контроль ошибок
- Физические сигналы, кабели, разъемы
- Ethernet, Wi-Fi, PPP, оптоволокно

### 2. Сетевой уровень (Internet Layer)
- Маршрутизация между сетями
- IP-адреса, пакеты, маршрутизация
- IP, ICMP, ARP, протоколы маршрутизации

### 3. Транспортный уровень (Transport Layer)
- Сквозная связь между приложениями
- TCP, UDP, порты, надежность
- Управление потоком данных

### 4. Прикладной уровень (Application Layer)
- Сетевые службы и приложения
- HTTP, FTP, SMTP, DNS, SSH
- Интерфейс для пользователей

## Сравнение с моделью OSI

| TCP/IP | OSI | Описание |
|--------|-----|----------|
| 4. Прикладной | 7. Прикладной | Сетевые службы |
| 4. Прикладной | 6. Представления | Форматы данных |
| 4. Прикладной | 5. Сеансовый | Управление сессиями |
| 3. Транспортный | 4. Транспортный | Сквозная связь |
| 2. Сетевой | 3. Сетевой | Маршрутизация |
| 1. Канальный | 2. Канальный | Передача между узлами |
| 1. Канальный | 1. Физический | Физические сигналы |

## Процессы передачи данных

### Инкапсуляция (Encapsulation)
1. **Прикладной уровень** - данные приложения
2. **Транспортный уровень** - добавляет TCP/UDP заголовок
3. **Сетевой уровень** - добавляет IP заголовок
4. **Канальный уровень** - добавляет Ethernet заголовок и передает биты

### Декапсуляция (Decapsulation)
Обратный процесс удаления заголовков при получении данных.

## Основные протоколы

### Стек протоколов TCP/IP
- **IP** - маршрутизация пакетов
- **TCP** - надежная передача данных
- **UDP** - быстрая передача данных
- **HTTP/HTTPS** - веб-протоколы
- **FTP** - передача файлов
- **SMTP** - электронная почта
- **DNS** - разрешение имен
- **SSH** - удаленный доступ

## Преимущества модели TCP/IP
- **Простота** - меньше уровней абстракции
- **Практичность** - реально используемая модель
- **Гибкость** - поддержка различных технологий
- **Совместимость** - работает с любыми средами передачи
- **Масштабируемость** - от локальных до глобальных сетей

## Применение
- **Интернет** - основа глобальной сети
- **Корпоративные сети** - внутренние сети организаций
- **Локальные сети** - офисные и домашние сети
- **Мобильные сети** - сотовые и Wi-Fi сети
- **Облачные сервисы** - распределенные вычисления
