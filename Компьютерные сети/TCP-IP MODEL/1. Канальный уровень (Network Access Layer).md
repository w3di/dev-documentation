# 1. Канальный уровень (Network Access Layer)

## Описание
Канальный уровень модели TCP/IP (Network Access Layer) объединяет функции физического и канального уровней модели OSI, обеспечивая надежную передачу данных между соседними узлами сети через физический канал связи.

## Основные функции
- **Формирование кадров** - структурирование данных в кадры с заголовками и концевиками
- **Адресация на канальном уровне** - использование MAC-адресов для идентификации устройств
- **Обнаружение и исправление ошибок** - контроль целостности данных через CRC и другие методы
- **Управление доступом к среде (MAC)** - контроль доступа к разделяемой среде передачи
- **Управление потоком данных** - предотвращение переполнения буферов получателя

## Структура данных
- **Frame (Кадр)** - единица передачи данных, содержащая заголовок, полезные данные и контрольную сумму
- **MAC-адреса** - 48-битные (6 байт) уникальные идентификаторы сетевых интерфейсов
- **Заголовок кадра** - содержит адреса отправителя и получателя, тип протокола, контрольную информацию
- **Концевик кадра** - содержит контрольную сумму (FCS) для обнаружения ошибок

## Протоколы канального уровня
- **Ethernet** - IEEE 802.3, основной протокол локальных сетей
- **Wi-Fi** - IEEE 802.11, беспроводные локальные сети
- **PPP** (Point-to-Point Protocol) - для прямых соединений
- **HDLC** (High-Level Data Link Control) - для синхронных соединений
- **Frame Relay** - для глобальных сетей

## Сетевые устройства
- **Коммутаторы (Switches)** - устройства для передачи кадров на основе MAC-адресов
- **Мосты (Bridges)** - устройства для соединения сегментов сети
- **Точки доступа (Access Points)** - для беспроводных сетей

## Характеристики
- **Локальная область действия** - работает в пределах одного сегмента сети (broadcast domain)
- **Физическая адресация** - использует MAC-адреса для идентификации устройств в локальной сети
- **Надежная доставка** - обеспечивает безошибочную передачу данных между соседними узлами
- **Дуплексная связь** - поддерживает одновременную передачу в обе стороны
- **Протокол-независимость** - может передавать данные различных сетевых протоколов
