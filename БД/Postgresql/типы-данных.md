## Типы данных для колонок таблиц

В SQL каждый столбец в таблице должен иметь определённый тип данных, указывающий на то, какая информация может храниться в этом столбце.

Тип данных столбца определяет:
- Какие операции можно выполнять с данными в этом столбце
- Сколько места требуется для хранения данных
- Как данные хранятся внутри

## Основные типы данных в SQL

### Строковый тип данных

Используется для хранения текстовых значений.

Примеры:
- `VARCHAR(n)` — строка переменной длины до n символов
- `TEXT` — текст неограниченной длины
- `CHAR(n)` — строка фиксированной длины n символов

### Числовой тип данных

Используется для хранения числовых значений.

Примеры:
- `INTEGER` — целое число
- `BIGINT` — большое целое число
- `NUMERIC` — число с фиксированной точностью
- `DECIMAL` — десятичное число
- `REAL` — число с плавающей точкой

### Булевый тип данных

Используется для хранения логических значений.

Примеры:
- `BOOLEAN` — `TRUE`, `FALSE` или `NULL`

### Дата и время

Используется для хранения значений даты и времени.

Примеры:
- `DATE` — дата (год, месяц, день)
- `TIME` — время (часы, минуты, секунды)
- `TIMESTAMP` — дата и время
- `TIMESTAMPTZ` — дата и время с временной зоной

## Важность выбора правильного типа данных

При определении таблицы в SQL важно выбрать соответствующие типы данных для каждого столбца, чтобы обеспечить:

Эффективное хранение данных:
- Правильный тип занимает оптимальное количество памяти
- Избыточные типы данных приводят к неэффективному использованию пространства

Эффективное извлечение данных:
- СУБД оптимизирует запросы на основе типов данных
- Индексы работают эффективнее с правильными типами

Целостность данных:
- Тип данных предотвращает некорректные значения
- Обеспечивает валидацию на уровне базы данных

## Рекомендации по выбору типов данных

- Учитывайте возможный диапазон значений, которые будут храниться в каждом столбце
- Выбирайте тип данных, соответствующий этому диапазону
- Не используйте избыточно большие типы данных
- Для строк с фиксированной длиной используйте `CHAR`, для переменной — `VARCHAR`
- Для больших текстов используйте `TEXT`
- Для денежных значений используйте `NUMERIC` или `DECIMAL` вместо типов с плавающей точкой

## Примеры использования

```sql
CREATE TABLE Products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    price NUMERIC(10, 2) NOT NULL,
    quantity INTEGER NOT NULL DEFAULT 0,
    is_available BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

В этом примере:
- `id` — автоинкрементное целое число
- `name` — строка до 255 символов
- `description` — текст неограниченной длины
- `price` — десятичное число с 10 цифрами, 2 из которых после запятой
- `quantity` — целое число
- `is_available` — булево значение
- `created_at` — дата и время создания записи
