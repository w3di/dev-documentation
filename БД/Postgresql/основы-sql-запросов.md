## Базовый синтаксис запроса

SELECT — выборка данных из таблицы

```sql
SELECT * FROM FamilyMembers
```
Выбирает все поля из таблицы FamilyMembers

```sql
SELECT member_id, member_name FROM FamilyMembers
```
Выбирает указанные поля из таблицы FamilyMembers

Псевдонимы:

AS создаёт псевдонимы для столбцов и таблиц

```sql
SELECT member_id, member_name AS Name FROM FamilyMembers
или
SELECT member_id, member_name Name FROM FamilyMembers
```
Переименовывает member_name в Name в результате запроса

## Литералы в SQL

Литерал — явно заданное фиксированное значение в запросе.

Типы литералов:
- строковые
- числовые
- логические
- NULL
- дата и время

Строковые литералы:
- Записываются в одинарных кавычках: 'SQL'
- Двойные кавычки " — только для идентификаторов
- Escape-последовательности (\n, \t и т.д.) работают только в E-строках

```sql
SELECT E'Строка\nДругая строка';
```
Пример использования E-строки для escape-последовательностей

Числовые литералы:
- Целые и дробные: 1, 2.5, .2
- Положительные и отрицательные: +1, -10
- Экспоненциальная форма: 1e3, 1e-3

Арифметические операторы: + - * / %

```sql
SELECT (5 * 2 - 6) / 2;
```
Пример арифметических операций в запросе

Литералы даты и времени:

Задаются строками или числами:
- Дата: '2020-01-01', '20200101'
- Время: '12:11', '121100'
- Дата и время: '2020-01-01 18:30:30', '20200101183030'

Разделители могут быть любыми или отсутствовать.

Логические литералы:
- TRUE
- FALSE

NULL:
- Означает отсутствие значения
- Не равно пустой строке и не равно 0
- Используется, когда данных нет вообще

## Встроенные функции SQL 

Встроенная функция — код внутри СУБД для обработки данных в запросе.
Принимают 0, 1 или несколько аргументов.
Всегда возвращают одно значение.

Примеры функций:
- UPPER(text) — верхний регистр
- LOWER(text) — нижний регистр
- LENGTH(text) — длина строки
- POSITION(sub IN str) — позиция подстроки (с 1)
- EXTRACT(part FROM date) — часть даты
- NOW() — текущая дата и время

Функции над полями таблицы:
- Применяются к значениям столбцов
- Выполняются для каждой строки отдельно
    
Комбинирование функций:
- Результат функции можно передавать в другую
    
```sql
UPPER(LEFT('sql-academy', 3))
```
Выводит первые 3 буквы в верхнем регистре

Арифметика над результатами функций:
- Результаты функций можно складывать, вычитать и т.д.
    
```sql
LENGTH(member_name) - POSITION(' ' IN member_name)
```
Из длины всего имени вычитает длину имени + пробела и получается длина фамилии

## DISTINCT — исключение дубликатов

DISTINCT убирает повторяющиеся строки из результата SELECT.
Применяется на этапе выборки данных.

```sql
SELECT DISTINCT поле FROM таблица;
```
Синтаксис для одной колонки

```sql
SELECT DISTINCT поле1, поле2 FROM таблица;
```
Синтаксис для нескольких колонок

Одна колонка:
- Убираются повторяющиеся значения одного поля.
    
```sql
SELECT DISTINCT class FROM Student_in_class;
```
Получает уникальные классы

Несколько колонок:
- Уникальность определяется по совокупности всех указанных колонок.
- Строки считаются дубликатами, если значения во всех колонках совпадают.
    
```sql
SELECT DISTINCT first_name, last_name FROM User;
```
В результате останутся все уникальные пары first_name + last_name
    
Важно:
- DISTINCT относится ко всему списку выбранных колонок, а не к каждой по отдельности.
