#### Основы HTML

1. Что такое HTML и для чего он используется?
	- HTML (HyperText Markup Language) — язык разметки, используемый для создания структуры веб-страниц

2. Что такое DOCTYPE и зачем он нужен?
	- DOCTYPE — объявление типа документа. Указывает браузеру, какую версию HTML использовать для интерпретации страницы. В HTML5 записывается как <!DOCTYPE html>

3. Какая структура у HTML документа?
	- HTML документ состоит из:
		1. DOCTYPE объявления
		2. Корневого элемента <html>
		3. Секции <head> с метаданными
		4. Секции <body> с содержимым страницы
		
		```html
		<!DOCTYPE html>
		<html lang="ru">
		<head>
			<meta charset="UTF-8">
			<title>Заголовок страницы</title>
		</head>
		<body>
			<!-- Содержимое страницы -->
		</body>
		</html>
		```

4. Что такое теги и атрибуты в HTML?
	- Теги — элементы разметки, которые определяют структуру и семантику содержимого
	- Атрибуты — дополнительные свойства тегов, которые предоставляют дополнительную информацию

5. Разница между блочными и строчными элементами в HTML?
	- Блочные элементы (<div>, <p>, <h1>, <section>):
		- Занимают всю ширину родительского элемента
		- Начинаются с новой строки
		- Могут содержать другие блочные и строчные элементы
		- Имеют отступы (margin) сверху и снизу
	- Строчные элементы (<span>, <a>, <strong>, <em>):
		- Занимают только необходимую ширину
		- Не начинаются с новой строки
		- Могут содержать только текст и другие строчные элементы
		- Не имеют отступов сверху и снизу

6. Что такое валидный HTML?
	- Валидный HTML — код, который соответствует стандартам W3C и правилам синтаксиса HTML
		- Правильная структура тегов
		- Корректное использование атрибутов
		- Соответствие DOCTYPE
		- Правильное закрытие всех тегов
		- Уникальные ID элементов

7. Как проверить валидность HTML документа?
	- Онлайн валидаторы: W3C Markup Validator (validator.w3.org), HTML Validator (html5.validator.nu)
	- Браузерные инструменты: DevTools в Chrome/Firefox, консоль браузера показывает ошибки
	- IDE и редакторы: встроенные валидаторы в VS Code, WebStorm, плагины для автоматической проверки
	- Командная строка: HTML Tidy, W3C validator API
	- Инструменты для React: eslint-plugin-jsx-a11y, @lhci/cli, axe-core


#### Семантика HTML

8. Что такое семантика и для чего она нужна?
	- Семантика — использование HTML тегов по их назначению для придания смысла содержимому
	- Цели семантики:
		- Улучшение доступности для скрин-ридеров
		- Лучшее SEO и индексация поисковиками
		- Более понятная структура кода
		- Автоматическое применение стилей браузером
		- Улучшение пользовательского опыта

9. Какие семантические теги существуют в HTML5?
	- Структурные теги: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>
	- Контентные теги: <figure>, <figcaption>, <time>, <mark>, <progress>, <meter>, <details>, <summary>
	- Интерактивные теги: <dialog>, <menu>, <menuitem>

10. Как работают <header>, <nav>, <main>, <section>, <article>?
	- <header> — шапка страницы или секции, содержит заголовки и навигацию
	- <nav> — навигационные ссылки (меню, breadcrumbs)
	- <main> — основное содержимое страницы (только один на странице)
	- <section> — тематическая секция с заголовком
	- <article> — самостоятельная статья или пост, который можно переиспользовать

11. Как работают <aside>, <footer>, <figure>, <figcaption>?
	- <aside> — боковая панель, дополнительная информация, реклама
	- <footer> — подвал страницы или секции с контактной информацией
	- <figure> — контейнер для изображений, диаграмм, кода с подписью
	- <figcaption> — подпись к содержимому <figure>

12. Как работают <time>, <mark>, <progress>, <meter>?
	- <time> — дата и время с атрибутом `datetime`
	- <mark> — выделенный текст (поисковые результаты, важная информация)
	- <progress> — прогресс выполнения задачи (0-100%)
	- <meter> — измерение в заданном диапазоне (диск, температура)


13. Как работают <details>, <summary>, <dialog>?
	- <details> — раскрывающийся блок с дополнительной информацией
	- <summary> — заголовок для <details>, по клику раскрывает/скрывает содержимое
	- <dialog> — модальное окно или диалог
	

14. Когда использовать <div> vs семантические теги?
	- Используйте <div> когда: нет подходящего семантического тега, нужен контейнер для стилизации, создаете layout без семантического значения
	- Используйте семантические теги когда: содержимое имеет четкое назначение, важна доступность и SEO, нужна автоматическая стилизация браузером
	- Правило: семантика > <div>, но не переусложняйте

15. В чем разница между <div> и <section>?
	- <div>: универсальный контейнер без семантического значения, используется для группировки элементов для стилизации, не влияет на структуру документа, не создает outline (план) страницы
	- <section>: семантический тег для тематических разделов, должен иметь заголовок (h1-h6), создает структуру документа, влияет на outline и доступность
	- Когда использовать <section>: группировка связанного контента, разделы статьи или страницы, контент, который можно выделить в отдельную секцию
	- Когда использовать <div>: контейнер для CSS стилизации, группировка без семантического значения, layout элементы (обертки, сетки)

16. Как работают заголовки <h1> - <h6>?
	- Иерархия заголовков от самого важного (`h1`) до наименее важного (`h6`)
	- <h1> — главный заголовок страницы (только один на странице)
	- <h2> - <h6> — подзаголовки разной важности
	- Особенности: влияют на SEO и структуру страницы, создают outline (план) документа, используются скрин-ридерами для навигации, имеют встроенные стили по умолчанию

17. Как работает иерархия заголовков?
	- Правила иерархии: не пропускайте уровни (h1 → h2 → h3, но не h1 → h3), один <h1> на страницу, логическая последовательность уровней
	- Структура outline: браузеры создают план документа на основе заголовков, скрин-ридеры используют для навигации, влияет на SEO и понимание структуры
	- Пример правильной иерархии:
		```html
		<h1>Главная страница</h1>
			<h2>Раздел 1</h2>
				<h3>Подраздел 1.1</h3>
				<h3>Подраздел 1.2</h3>
			<h2>Раздел 2</h2>
				<h3>Подраздел 2.1</h3>
					<h4>Детали 2.1.1</h4>
		```

#### Метатеги и head

18. Что такое метатеги и какие важные метатеги нужно использовать?
	- Метатеги — элементы <meta> в <head>, предоставляющие метаданные о странице
	- Обязательные метатеги: <meta charset="UTF-8"> (кодировка), <meta name="viewport"> (адаптивность), <title> (заголовок страницы)
	- SEO метатеги: <meta name="description"> (описание страницы), <meta name="keywords"> (ключевые слова), <meta name="robots"> (инструкции для поисковиков)
	- Социальные метатеги: Open Graph (Facebook), Twitter Cards, Schema.org

19. Как работает <title> и зачем он нужен?
	- <title> — заголовок страницы, отображается во вкладке браузера
	- Важность: основной фактор SEO, отображается в результатах поиска, помогает пользователям ориентироваться
	- Правила написания: 50-60 символов, уникальный для каждой страницы, описательный и привлекательный, включает ключевые слова

20. Как работают <meta charset>, <meta name="description">?
	- <meta charset="UTF-8">: указывает кодировку символов, должен быть первым в <head>, UTF-8 поддерживает все языки
	- <meta name="description">: описание страницы для поисковиков, отображается в сниппетах поиска, 150-160 символов, должно быть уникальным и релевантным

21. Как работают <meta name="viewport">, <meta name="robots">?
	- <meta name="viewport">: настройки отображения на мобильных устройствах, контролирует масштабирование и ширину, обязателен для адаптивного дизайна
	- <meta name="robots">: инструкции для поисковых роботов, контролирует индексацию и переходы по ссылкам

22. Как работают Open Graph метатеги?
	- Open Graph — протокол для красивого отображения ссылок в социальных сетях
	- Основные теги: `og:title` (заголовок), `og:description` (описание), `og:image` (изображение), `og:url` (URL страницы), `og:type` (тип контента)

23. Как работают Twitter Card метатеги?
	- Twitter Cards — специальные метатеги для Twitter
	- Типы карточек: `summary` (базовая карточка), `summary_large_image` (с большим изображением), `app` (для мобильных приложений)
	- Основные теги: `twitter:card` (тип карточки), `twitter:title` (заголовок), `twitter:description` (описание), `twitter:image` (изображение)


24. Как работает <link rel="canonical">?
	- Canonical URL — указывает поисковикам основную версию страницы
	- Используется для: предотвращения дублированного контента, объединения ссылочного веса, указания предпочтительной версии страницы

25. Как работает <link rel="preconnect">, <link rel="dns-prefetch">?
	- <link rel="preconnect">: устанавливает соединение с доменом заранее, включает DNS lookup, TCP handshake, TLS negotiation, ускоряет загрузку ресурсов с внешних доменов
	- <link rel="dns-prefetch">: только DNS lookup для домена, менее ресурсозатратный чем preconnect, подходит для доменов, которые могут не понадобиться

26. Как работают <link rel="stylesheet">, <link rel="icon">?
	- <link rel="stylesheet">: подключает внешние CSS файлы, может иметь атрибуты `media` и `title`, блокирует рендеринг до загрузки
	- <link rel="icon">: устанавливает иконку сайта (favicon), поддерживает различные форматы (ICO, PNG, SVG), может указывать размеры для разных устройств

27. Как работает <base> тег?
	- <base> — устанавливает базовый URL для всех относительных ссылок на странице
	- Атрибуты: `href` (базовый URL), `target` (целевое окно по умолчанию)
	- Особенности: должен быть в <head> перед другими ссылками, влияет на все относительные URL на странице, может быть только один на странице


#### Формы и инпуты

28. Как работают HTML формы?
	- HTML формы — элементы для сбора и отправки данных пользователя
	- Основные компоненты: <form> (контейнер для формы), <input> (поля ввода), <button> (кнопки отправки), <label> (подписи к полям)
	- Атрибуты формы: `action` (URL для отправки данных), `method` (HTTP метод GET/POST), `enctype` (тип кодировки данных), `target` (окно для отправки)

29. Какие типы input существуют в HTML5?
	- Текстовые поля: `text`, `email`, `password`, `search`, `tel`, `url`
	- Числовые поля: `number`, `range`
	- Дата и время: `date`, `time`, `datetime-local`, `month`, `week`
	- Выбор: `checkbox`, `radio`
	- Файлы и скрытые: `file`, `hidden`
	- Специальные: `color`, `submit`, `reset`, `button`

30. Как работают <input type="text">, <input type="email">, <input type="password">?
	- <input type="text">: однострочное текстовое поле, базовый тип для ввода текста, поддерживает `maxlength`, `pattern`, `placeholder`
	- <input type="email">: поле для ввода email адреса, автоматическая валидация формата email, мобильные устройства показывают email клавиатуру
	- <input type="password">: поле для ввода пароля, символы скрыты (заменены на точки/звездочки), не сохраняется в истории браузера

31. Как работают <input type="number">, <input type="tel">, <input type="url">?
	- <input type="number">: поле для ввода чисел, поддерживает `min`, `max`, `step`, мобильные устройства показывают цифровую клавиатуру
	- <input type="tel">: поле для ввода телефонного номера, мобильные устройства показывают телефонную клавиатуру, не валидирует формат автоматически
	- <input type="url">: поле для ввода URL, автоматическая валидация формата URL, должен начинаться с протокола ()

32. Как работают <input type="date">, <input type="time">, <input type="datetime-local">?
	- <input type="date">: поле для выбора даты, показывает календарь в поддерживающих браузерах, поддерживает `min`, `max` для ограничения диапазона
	- <input type="time">: поле для выбора времени, показывает часы и минуты, поддерживает `step` для точности (секунды)
	- <input type="datetime-local">: поле для выбора даты и времени, объединяет функциональность date и time, локальное время без часового пояса

33. Как работают <input type="checkbox">, <input type="radio">?
	- <input type="checkbox">: чекбокс для множественного выбора, можно выбрать несколько вариантов, значение отправляется только если отмечен
	- <input type="radio">: радиокнопка для единственного выбора, группируется по атрибуту `name`, только один вариант в группе может быть выбран
	- Особенности: оба типа используют атрибут `value`, `checked` (предварительно выбранный), `disabled` (неактивный элемент)

34. Как работают <input type="range">, <input type="color">?
	- <input type="range">: слайдер для выбора значения в диапазоне, поддерживает `min`, `max`, `step`, значение можно отображать через JavaScript
	- <input type="color">: поле для выбора цвета, показывает цветовой пикер, возвращает значение в формате HEX

35. Как работают <input type="file">, <input type="hidden">?
	- <input type="file">: поле для загрузки файлов, поддерживает `accept` для фильтрации типов файлов, `multiple` (выбор нескольких файлов)
	- <input type="hidden">: скрытое поле, не отображается пользователю, используется для передачи служебных данных, значение устанавливается через JavaScript или сервер


36. Как работает <select> и <option>?
	- <select> — выпадающий список для выбора одного или нескольких значений
	- <option> — отдельный вариант в списке
	- Атрибуты: `multiple` (множественный выбор), `size` (количество видимых вариантов), `disabled` (неактивный список)
	- Группировка: <optgroup> (группировка вариантов), `label` (заголовок группы)

37. Как работает <textarea>?
	- <textarea> — многострочное текстовое поле
	- Атрибуты: `rows` (количество строк), `cols` (количество столбцов), `maxlength` (максимальная длина), `placeholder` (подсказка)
	- Особенности: содержимое между тегами — начальное значение, можно изменять размер пользователем, поддерживает `resize` в CSS

38. Как работают <fieldset> и <legend>?
	- <fieldset> — группирует связанные поля формы
	- <legend> — заголовок для группы полей
	- Использование: улучшает доступность, визуально группирует поля, помогает скрин-ридерам

39. Как работают <label> и связь с input?
	- <label> — подпись к полю формы
	- Способы связи: `for` атрибут указывает на `id` поля, обертывание поля в <label>
	- Преимущества: клик по подписи активирует поле, улучшает доступность, помогает скрин-ридерам

40. Как работают <button> и <input type="submit">?
	- <button>: более гибкий элемент кнопки, может содержать HTML контент, типы: `submit`, `reset`, `button`
	- <input type="submit">: специализированная кнопка отправки, только текстовое содержимое, `value` (текст кнопки)
	- Различия: <button> более семантичен, <button> поддерживает вложенные элементы, <input> проще для базовых случаев


41. Как работают атрибуты `required`, `disabled`, `readonly`?
	- `required`: поле обязательно для заполнения, браузер показывает ошибку при отправке, валидация происходит автоматически
	- `disabled`: поле неактивно и не отправляется, нельзя изменить значение, не участвует в валидации
	- `readonly`: поле только для чтения, значение отправляется с формой, можно выделить и скопировать текст

42. Как работают `placeholder`, `maxlength`, `min`, `max`?
	- `placeholder`: подсказка в поле ввода, исчезает при вводе текста, не заменяет <label>
	- `maxlength`: максимальная длина текста, браузер не позволяет ввести больше, показывает счетчик символов
	- `min`/`max`: минимальное/максимальное значение, работает с `number`, `date`, `range`, валидация происходит автоматически

43. Как работает валидация форм в HTML5?
	- Автоматическая валидация: браузер проверяет поля при отправке, показывает сообщения об ошибках, блокирует отправку невалидной формы
	- Типы валидации: `required` (обязательные поля), `pattern` (регулярные выражения), `min`/`max` (числовые ограничения), `minlength`/`maxlength` (длина текста), `type` (формат данных email, url)
	- Кастомизация: CSS псевдоклассы `:valid`, `:invalid`, JavaScript API для проверки, собственные сообщения об ошибках


#### Медиа контент

44. Как работает <img> тег и его атрибуты?
	- <img> — тег для отображения изображений
	- Обязательные атрибуты: `src` (путь к изображению), `alt` (альтернативный текст)
	- Основные атрибуты: `width`, `height` (размеры изображения), `title` (всплывающая подсказка), `loading` (ленивая загрузка), `decoding` (способ декодирования)
	- Современные атрибуты: `srcset` (набор изображений для разных разрешений), `sizes` (размеры для разных экранов), `fetchpriority` (приоритет загрузки)

45. Как работают атрибуты тэга img такие как `src`, `alt`, `title`, `width`, `height`?
	- `src`: путь к файлу изображения, поддерживает относительные и абсолютные URL, обязательный атрибут
	- `alt`: альтернативный текст для доступности, отображается при ошибке загрузки, используется скрин-ридерами, обязательный атрибут
	- `title`: всплывающая подсказка при наведении, необязательный атрибут, не заменяет `alt`
	- `width` и `height`: задают размеры изображения, предотвращают layout shift, рекомендуется указывать для производительности

46. Как работает <picture> и <source>?
	- <picture> — контейнер для адаптивных изображений
	- <source> — определяет разные версии изображения
	- Атрибуты <source>: `srcset` (набор изображений), `media` (медиа-запросы), `type` (MIME-тип изображения), `sizes` (размеры для разных экранов)
	- Использование: адаптивные изображения, поддержка современных форматов (WebP, AVIF), разные изображения для разных устройств

47. Как работает <video> и его атрибуты?
	- <video> — элемент для воспроизведения видео
	- Основные атрибуты: `src` (путь к видеофайлу), `controls` (показывать элементы управления), `autoplay` (автоматическое воспроизведение), `muted` (воспроизведение без звука), `loop` (зацикливание), `poster` (изображение-превью)
	- Размеры и производительность: `width`, `height` (размеры плеера), `preload` (стратегия предзагрузки), `loading` (ленивая загрузка)
	- Доступность: <track> (субтитры и титры), `kind="captions"` (субтитры), `kind="descriptions"` (описания)

48. Как работает <audio> и его атрибуты?
	- <audio> — элемент для воспроизведения аудио
	- Основные атрибуты: `src` (путь к аудиофайлу), `controls` (показывать элементы управления), `autoplay` (автоматическое воспроизведение), `muted` (воспроизведение без звука), `loop` (зацикливание)
	- Производительность: `preload` (стратегия предзагрузки), `loading` (ленивая загрузка)
	- Форматы: MP3 (универсальная поддержка), OGG (открытый формат), WAV (несжатый формат)

49. Как работают <iframe> и их безопасность?
	- <iframe> — встраивает внешний контент на страницу
	- Основные атрибуты: `src` (URL встраиваемого контента), `width`, `height` (размеры фрейма), `title` (описание для доступности), `name` (имя фрейма)
	- Безопасность: `sandbox` (ограничения безопасности), `allow` (разрешенные функции), `referrerpolicy` (политика referrer), `loading` (ленивая загрузка)
	- Атрибуты безопасности: `sandbox="allow-scripts allow-same-origin"`, `allow="camera; microphone"`, `referrerpolicy="no-referrer"`

50. Как работает <embed> и <object>?
	- <embed>: встраивает внешний контент (плагины, Flash), устаревший элемент, не рекомендуется, используется для Flash, PDF, Java-апплетов
	- <object>: более гибкий элемент для встраивания, поддерживает fallback контент, может содержать <param> для параметров
	- Современные альтернативы: <iframe> для веб-контента, <video> и <audio> для медиа, <canvas> для графики

51. Как работает <svg> в HTML?
	- <svg> — масштабируемая векторная графика
	- Преимущества: масштабируется без потери качества, маленький размер файла, поддержка анимации и интерактивности, доступность через ARIA
	- Основные элементы: <rect> (прямоугольник), <circle> (круг), <path> (сложные формы), <text> (текст), <g> (группа элементов)
	- Атрибуты: `viewBox` (система координат), `width`, `height` (размеры), `fill`, `stroke` (заливка и обводка)

52. Как работает <canvas> элемент?
	- <canvas> — HTML элемент для программного рисования графики
	- Основные атрибуты: `width`, `height` (размеры холста в пикселях), `id` (идентификатор для JavaScript)
	- Особенности: размеры задаются атрибутами, не CSS, по умолчанию прозрачный, требует JavaScript для рисования, поддерживает 2D и 3D (WebGL) контексты

#### Ссылки и навигация

53. Как работает <a> тег и его атрибуты?
	- <a> — тег для создания гиперссылок
	- Основные атрибуты: `href` (URL или якорь для перехода), `target` (окно для открытия ссылки), `rel` (отношения между документами), `title` (всплывающая подсказка), `download` (принудительная загрузка файла)
	- Особенности: может содержать текст, изображения, другие элементы, поддерживает различные протоколы (http, https, mailto, tel), влияет на SEO и доступность

54. Как работают `href`, `target`, `rel` атрибуты?
	- `href`: URL страницы или ресурса, якорные ссылки (#section), относительные и абсолютные пути, протоколы: http, https, mailto, tel, ftp
	- `target`: `_self` (текущее окно по умолчанию), `_blank` (новое окно/вкладка), `_parent` (родительское окно), `_top` (верхнее окно), имя окна (конкретное окно)
	- `rel`: `nofollow` (не переходить по ссылке), `noopener` (безопасность для _blank), `noreferrer` (не передавать referrer), `external` (внешняя ссылка), `canonical` (каноническая ссылка)

55. Как работают якорные ссылки?
	- Якорные ссылки — переходы к определенным местам на странице
	- Создание якоря: <a id="section1">Заголовок</a>, <h2 id="section1">Заголовок</h2>
	- Ссылка на якорь: <a href="#section1">Перейти к разделу</a>, <a href="page.html#section1">Раздел на другой странице</a>
	- Особенности: плавная прокрутка с CSS `scroll-behavior: smooth`, URL обновляется с якорем, работает с JavaScript для динамических переходов

56. Как работают `mailto:` и `tel:` ссылки?
	- `mailto:` ссылки: <a href="mailto:user@example.com">Написать письмо</a>, параметры: `subject`, `body`, `cc`, `bcc`, <a href="mailto:user@example.com?subject=Тема&body=Текст">Письмо с темой</a>
	- `tel:` ссылки: <a href="tel:+1234567890">Позвонить</a>, работает на мобильных устройствах, формат: `tel:+код_страны_номер`
	- Преимущества: улучшают пользовательский опыт, работают на всех устройствах, интегрируются с системными приложениями

57. Как работает <nav> для навигации?
	- <nav> — семантический тег для навигационных ссылок
	- Использование: главное меню сайта, breadcrumbs (хлебные крошки), пагинация, оглавление страницы
	- Структура:
		```html
		<nav>
			<ul>
				<li><a href="/">Главная</a></li>
				<li><a href="/about">О нас</a></li>
				<li><a href="/contact">Контакты</a></li>
			</ul>
		</nav>
		```
	- Преимущества: улучшает доступность, помогает скрин-ридерам, семантически правильная структура

58. Как работают breadcrumbs в HTML?
	- Breadcrumbs — навигационная цепочка, показывающая путь к текущей странице
	- Структура:
		```html
		<nav aria-label="Breadcrumb">
			<ol>
				<li><a href="/">Главная</a></li>
				<li><a href="/category">Категория</a></li>
				<li aria-current="page">Текущая страница</li>
			</ol>
		</nav>
		```
	- Атрибуты доступности: `aria-label="Breadcrumb"` (описание навигации), `aria-current="page"` (текущая страница)
	- Стилизация: обычно с разделителями (/, >, →), CSS для оформления списка, адаптивный дизайн для мобильных

#### Таблицы

59. Как работают HTML таблицы?
	- HTML таблицы — структурированный способ отображения данных в строках и столбцах
	- Основные элементы: <table> (контейнер таблицы), <tr> (строка таблицы), <td> (ячейка с данными), <th> (заголовочная ячейка)
	- Структурные элементы: <thead> (заголовок таблицы), <tbody> (тело таблицы), <tfoot> (подвал таблицы)
	- Особенности: автоматическое выравнивание ячеек, поддержка объединения ячеек, семантическая структура для доступности

60. Как работают <table>, <tr>, <td>, <th>?
	- <table>: корневой элемент таблицы, может содержать атрибуты `border`, `cellpadding`, `cellspacing`, поддерживает CSS стили для оформления
	- <tr>: определяет строку таблицы, содержит ячейки <td> или <th>, поддерживает атрибуты выравнивания
	- <td>: ячейка с обычными данными, атрибуты: `colspan`, `rowspan`, `headers`, может содержать любой HTML контент
	- <th>: заголовочная ячейка, по умолчанию жирный шрифт и центрирование, атрибут `scope` для связи с данными

61. Как работают <thead>, <tbody>, <tfoot>?
	- <thead>: заголовочная секция таблицы, содержит заголовочные строки, повторяется на каждой странице при печати
	- <tbody>: основное тело таблицы, содержит строки с данными, может быть несколько <tbody> в одной таблице
	- <tfoot>: подвал таблицы, содержит итоговые строки, отображается внизу таблицы
	- Порядок элементов:
		```html
		<table>
			<thead>...</thead>
			<tfoot>...</tfoot>
			<tbody>...</tbody>
		</table>
		```

62. Как работают `colspan` и `rowspan`?
	- `colspan`: объединяет ячейки по горизонтали, значение — количество объединяемых столбцов, <td colspan="2">Объединенная ячейка</td>
	- `rowspan`: объединяет ячейки по вертикали, значение — количество объединяемых строк, <td rowspan="2">Объединенная ячейка</td>

63. Как работает <caption> для таблиц?
	- <caption> — заголовок таблицы
	- Расположение: должен быть первым элементом в <table>, отображается над таблицей по умолчанию
	- Использование:
		```html
		<table>
			<caption>Продажи за месяц</caption>
			<thead>...</thead>
			<tbody>...</tbody>
		</table>
		```
	- Преимущества: улучшает доступность, помогает скрин-ридерам, семантически описывает содержимое таблицы

64. Как работают <col> и <colgroup>?
	- <colgroup>: группирует столбцы для стилизации, содержит элементы <col>, позволяет применять стили к целым столбцам
	- <col>: определяет свойства столбца, атрибуты: `span`, `width`, `align`, не содержит контент, только стили
	- Пример использования:
		```html
		<table>
			<colgroup>
				<col style="background-color: lightblue;">
				<col span="2" style="background-color: lightgreen;">
			</colgroup>
			<thead>...</thead>
			<tbody>...</tbody>
		</table>
		```

#### Списки

65. Как работают <ul>, <ol>, <li>?
	- <ul> (Unordered List): неупорядоченный список, элементы отмечены маркерами, используется для перечислений без порядка
	- <ol> (Ordered List): упорядоченный список, элементы пронумерованы, используется для последовательностей
	- <li> (List Item): элемент списка, может содержать любой HTML контент, поддерживает вложенные списки

66. Как работают <dl>, <dt>, <dd>?
	- <dl> (Description List): список описаний, содержит пары терминов и их определений, используется для глоссариев, FAQ, метаданных
	- <dt> (Description Term): термин для описания, обычно краткий заголовок
	- <dd> (Description Data): описание термина, может быть длинным текстом, может быть несколько <dd> для одного <dt>
	- Пример:
		```html
		<dl>
			<dt>HTML</dt>
			<dd>Язык разметки гипертекста</dd>
			
			<dt>CSS</dt>
			<dd>Каскадные таблицы стилей</dd>
			<dd>Используется для оформления веб-страниц</dd>
		</dl>
		```

67. Как работают вложенные списки?
	- Вложенность списков: списки могут содержать другие списки, создает иерархическую структуру, каждый уровень имеет свои маркеры/номера
	- Особенности: браузеры автоматически меняют маркеры, CSS позволяет кастомизировать внешний вид, важно для семантической структуры

68. Как работают атрибуты `type` и `start` для списков?
	- Атрибут `type` для <ul>: `disc` (закрашенный круг по умолчанию), `circle` (пустой круг), `square` (квадрат), `none` (без маркера)
	- Атрибут `type` для <ol>: `1` (арабские цифры по умолчанию), `a` (строчные буквы), `A` (заглавные буквы), `i` (строчные римские цифры), `I` (заглавные римские цифры)
	- Атрибут `start` для <ol>: задает начальное значение нумерации, <ol start="5"> (начнет с 5)

#### Доступность (a11y)

69. Как сделать доступную (a11y) верстку?
	- Использование семантических HTML тегов (<header>, <nav>, <main>, <section>, <article>)
	- Правильная структура заголовков (h1-h6) без пропуска уровней
	- Альтернативный текст для изображений (`alt` атрибут)
	- Достаточный цветовой контраст (минимум 4.5:1 для обычного текста)
	- Навигация с клавиатуры (`tabindex`, `taborder`)
	- ARIA атрибуты для сложных интерактивных элементов
	- Тестирование со скрин-ридерами

70. Как работают ARIA атрибуты?
	- ARIA (Accessible Rich Internet Applications) — набор атрибутов для улучшения доступности
	- Расширяет семантику HTML для скрин-ридеров и других вспомогательных технологий
	- Основные категории: роли (`role`), свойства (`aria-*`), состояния (`aria-*`)
	- Не заменяет семантические HTML теги, а дополняет их
	- Используется для сложных UI компонентов (модальные окна, выпадающие меню, табы)

71. Как работают `role`, `aria-label`, `aria-describedby`?
	- `role`: определяет роль элемента для скрин-ридеров (`button`, `dialog`, `menu`, `tablist`)
	- `aria-label`: предоставляет доступное имя для элемента, когда видимый текст недостаточен
	- `aria-describedby`: связывает элемент с дополнительным описанием через ID другого элемента
	- Пример: <button aria-label="Закрыть модальное окно" aria-describedby="close-help">×</button>

72. Как работают `aria-hidden`, `aria-expanded`, `aria-selected`?
	- `aria-hidden="true"`: скрывает элемент от скрин-ридеров (декоративные элементы)
	- `aria-expanded`: указывает, развернут ли элемент (выпадающие меню, аккордеоны)
	- `aria-selected`: указывает, выбран ли элемент в списке или группе
	- Пример: <button aria-expanded="false" aria-controls="menu">Меню</button>

73. Как работают `tabindex` и навигация с клавиатуры?
	- `tabindex="0"`: элемент участвует в навигации с клавиатуры в естественном порядке
	- `tabindex="-1"`: элемент может получить фокус программно, но не участвует в tab-навигации
	- `tabindex="1+"`: элемент получает приоритет в навигации (не рекомендуется)
	- Навигация: Tab (следующий), Shift+Tab (предыдущий), Enter/Space (активация)

74. Как работают `alt` атрибуты для изображений?
	- `alt=""`: декоративные изображения, которые не несут смысловой нагрузки
	- `alt="описание"`: информативные изображения с описанием содержимого
	- `alt` отсутствует: изображение считается декоративным (не рекомендуется)
	- Длинные описания: использовать `aria-describedby` для связи с подробным описанием

75. Как работают `title` атрибуты?
	- `title`: всплывающая подсказка при наведении мыши
	- Не заменяет `alt` для изображений
	- Недоступен для пользователей клавиатуры и сенсорных устройств
	- Используется для дополнительной информации, но не критичной для понимания

76. Как работают `lang` атрибуты?
	- `lang="ru"`: указывает язык содержимого страницы или элемента
	- Помогает скрин-ридерам правильно произносить текст
	- Влияет на направление текста (RTL для арабского, иврита)
	- Пример: <html lang="ru">, <span lang="en">Hello</span>

77. Как работают skip links?
	- Skip links — ссылки для быстрого перехода к основному контенту
	- Помогают пользователям клавиатуры пропустить навигацию
	- Обычно скрыты, появляются при получении фокуса
	- Пример: <a href="#main" class="skip-link">Перейти к основному контенту</a>

78. Как работают landmarks в ARIA?
	- Landmarks — семантические области страницы для навигации
	- Основные роли: `banner` (шапка), `navigation` (навигация), `main` (основной контент), `complementary` (боковая панель), `contentinfo` (подвал)
	- Автоматически создаются семантическими тегами:
		- <header> → `banner` (если не внутри <article> или <section>)
		- <nav> → `navigation`
		- <main> → `main`
		- <aside> → `complementary`
		- <footer> → `contentinfo` (если не внутри <article> или <section>)
	- Можно добавлять вручную: <div role="banner"> или <div role="main">

79. Как работают `aria-live` и `aria-atomic`?
	- `aria-live` — уведомляет скрин-ридеры об изменениях в динамическом контенте
	- Значения:
		- `aria-live="off"` — не уведомлять (по умолчанию)
		- `aria-live="polite"` — уведомить после завершения текущего действия
		- `aria-live="assertive"` — уведомить немедленно (прерывает текущее чтение)
	- `aria-atomic` — контролирует, что читать при изменении:
		- `aria-atomic="false"` — читать только изменившуюся часть
		- `aria-atomic="true"` — читать весь контейнер целиком
	- Примеры использования:
		- Счетчик товаров: <span aria-live="polite">В корзине 3 товара</span>
		- Уведомления: <div aria-live="assertive" aria-atomic="true">Ошибка: неверный пароль</div>
		- Статус загрузки: <div aria-live="polite" aria-atomic="false">Загружено 50%</div>

#### Микроразметка

80. Что такое микроразметка и зачем она нужна?
	- Микроразметка — структурированные данные, которые помогают поисковикам понимать содержимое страницы
	- Улучшает отображение в результатах поиска (rich snippets, карточки, звездочки)
	- Повышает CTR и рейтинг в поисковой выдаче
	- Помогает поисковикам создавать knowledge graph
	- Основные форматы: JSON-LD, Microdata, RDFa

81. Как работает JSON-LD микроразметка?
	- JSON-LD (JSON for Linked Data) — современный формат структурированных данных
	- Размещается в <script type="application/ld+json"> в <head> или <body>
	- Не влияет на отображение страницы, только на SEO
	- Легко читается и редактируется
	- Пример: <script type="application/ld+json">{"@context": "", "@type": "Article"}</script>

82. Как работает Schema.org разметка?
	- Schema.org — стандартизированная схема для структурированных данных
	- Создана совместно Google, Microsoft, Yahoo, Yandex
	- Содержит типы: Article, Product, Event, Organization, Person, Review
	- Каждый тип имеет обязательные и рекомендуемые свойства
	- Поддерживается всеми крупными поисковиками
	- Пример полной микроразметки для статьи:
		```html
		<script type="application/ld+json">
		{
			"@context": "",
			"@type": "Article",
			"headline": "Как создать доступный сайт",
			"description": "Подробное руководство по созданию доступных веб-сайтов",
		```

83. Как работает микроразметка для статей?
	- Тип: `Article` или `NewsArticle`
	- Обязательные свойства: `headline`, `author`, `datePublished`, `dateModified`
	- Рекомендуемые: `image`, `description`, `publisher`, `mainEntityOfPage`
	- Помогает отображаться в Google News и rich snippets
	- Пример: статья с автором, датой публикации, изображением

84. Как работает микроразметка для товаров?
	- Тип: `Product`
	- Обязательные свойства: `name`, `description`, `image`
	- Рекомендуемые: `brand`, `sku`, `offers` (цена, валюта, доступность), `aggregateRating` (рейтинг)
	- Отображается в Google Shopping и rich snippets с ценой
	- Поддерживает отзывы и рейтинги

85. Как работает микроразметка для событий?
	- Тип: `Event`
	- Обязательные свойства: `name`, `startDate`, `location`
	- Рекомендуемые: `description`, `endDate`, `organizer`, `offers` (билеты)
	- Отображается в Google Events и календарных приложениях
	- Помогает пользователям найти события рядом

	- Обязательные свойства: `name`, `url`
	- Рекомендуемые: `logo`, `description`, `address`, `telephone`, `openingHours`, `sameAs` (соцсети)
	- Отображается в knowledge panel в Google
	- Помогает с локальным SEO

#### Производительность

87. Как оптимизировать HTML для производительности?
	- Минификация HTML (удаление пробелов, комментариев, переносов строк)
	- Сжатие с помощью gzip/brotli:
		- gzip: сжимает HTML на 60-80%, поддерживается всеми браузерами
		- brotli: сжимает на 15-25% лучше gzip, поддерживается современными браузерами
		- Настраивается на сервере (nginx, apache) или CDN
	- Оптимизация порядка загрузки ресурсов (critical CSS в <head>)
	- Использование `loading="lazy"` для изображений
	- Preloading критических ресурсов (<link rel="preload">)
	- DNS prefetch для внешних доменов:
		- <link rel="dns-prefetch" href="//fonts.googleapis.com">
		- <link rel="dns-prefetch" href="//cdn.example.com">
		- Ускоряет загрузку ресурсов с внешних доменов на 200-500ms
		- Используйте для Google Fonts, CDN, аналитики, виджетов
	- Уменьшение размера DOM (меньше вложенности)

88. Как работает lazy loading для изображений?
	- `loading="lazy"`: изображения загружаются только при приближении к viewport
	- Экономит трафик и ускоряет первоначальную загрузку страницы
	- Поддерживается современными браузерами
	- Fallback для старых браузеров через Intersection Observer API
	- Не рекомендуется для изображений выше сгиба (above the fold):
		- Above the fold — контент, видимый без прокрутки при загрузке страницы
		- Логотип, главное изображение, навигация должны загружаться сразу
		- Используйте `loading="eager"` или без атрибута для критических изображений
	- Пример: <img src="image.jpg" loading="lazy" alt="Описание">

89. Как работает preloading ресурсов?
	- <link rel="preload">: загружает критически важные ресурсы заранее
	- Используется для шрифтов, CSS, JavaScript, изображений
	- Атрибуты: `href` (путь), `as` (тип ресурса), `type` (MIME-тип)
	- Примеры: <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
	- Отличается от <link rel="prefetch"> (загружает для следующей страницы)

90. Как работает critical CSS в HTML?
	- Critical CSS — стили, необходимые для отображения above-the-fold контента
	- Встраивается в <head> для избежания блокировки рендеринга
	- Остальные стили загружаются асинхронно
	- Уменьшает FCP (First Contentful Paint) и LCP (Largest Contentful Paint)
	- Автоматически генерируется инструментами (Critical, Penthouse)

91. Как работает resource hints?
	- <link rel="dns-prefetch">: предварительный DNS lookup для домена
	- <link rel="preconnect">: устанавливает соединение с доменом (DNS + TCP + TLS)
	- <link rel="prefetch">: загружает ресурс для следующей страницы
	- <link rel="prerender">: предварительно рендерит страницу
	- Ускоряют загрузку внешних ресурсов и навигацию

#### Безопасность

92. Как обеспечить безопасность HTML?
	- Валидация и санитизация пользовательского ввода
	- Использование HTTPS для всех ресурсов
	- Content Security Policy (CSP) для предотвращения XSS
	- Безопасные атрибуты для ссылок (`rel="noopener"`, `rel="noreferrer"`)
	- Sandbox для iframe с ограниченными правами
	- Защита от clickjacking через X-Frame-Options
	- Регулярное обновление зависимостей

93. Как работает Content Security Policy?
	- CSP — механизм безопасности для предотвращения XSS атак
	- Указывает, какие ресурсы может загружать страница
	- Директивы: `default-src`, `script-src`, `style-src`, `img-src`, `connect-src`
	- Устанавливается через HTTP заголовок или <meta> тег
	- Пример: `Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'`

94. Как защититься от XSS атак?
	- Экранирование HTML символов (<`, `>, `&`, `"`, `'`)
	- Использование `textContent` вместо `innerHTML` в JavaScript
	- Валидация и санитизация пользовательского ввода
	- Content Security Policy для ограничения выполнения скриптов
	- Использование безопасных методов рендеринга (React, Vue автоматически экранируют)
	- HttpOnly флаги для cookies

95. Как работают `rel="noopener"` и `rel="nofollow"`?
	- `rel="noopener"`: предотвращает доступ нового окна к `window.opener` (защита от tabnabbing)
	- `rel="noreferrer"`: не передает referrer информацию (дополнительная защита)
	- `rel="nofollow"`: указывает поисковикам не переходить по ссылке (SEO)
	- Обязательно для ссылок с `target="_blank"`
	- Пример: <a href="" target="_blank" rel="noopener noreferrer">Ссылка</a>

96. Как работает `sandbox` атрибут для iframe?
	- `sandbox`: ограничивает возможности iframe для безопасности
	- Значения: `allow-scripts`, `allow-same-origin`, `allow-forms`, `allow-popups`
	- По умолчанию блокирует все: скрипты, формы, навигацию, popup'ы
	- Защищает от malicious iframe контента
	- Пример: <iframe src="untrusted.html" sandbox="allow-scripts allow-same-origin"></iframe>

#### SEO

97. Как оптимизировать HTML для SEO?
	- Семантическая разметка с правильными HTML тегами
	- Иерархическая структура заголовков (h1-h6)
	- Уникальные и описательные title и meta description
	- Альтернативный текст для изображений
	- Структурированные данные (Schema.org)
	- Быстрая загрузка страницы (Core Web Vitals)
	- Мобильная адаптивность
	- Внутренняя перелинковка

98. Как работают заголовки для SEO?
	- H1: главный заголовок страницы (только один), содержит основное ключевое слово
	- H2-H6: подзаголовки, создают иерархию контента
	- Не пропускать уровни (h1 → h2 → h3, но не h1 → h3)
	- Заголовки должны описывать содержимое секции
	- Влияют на понимание структуры страницы поисковиками
	- Используются для создания table of contents в результатах поиска

99. Как работают мета-описания?
	- <meta name="description">: описание страницы для поисковиков
	- Отображается в сниппетах поисковой выдачи
	- Длина: 150-160 символов (оптимально)
	- Должно быть уникальным для каждой страницы
	- Включать призыв к действию (CTA)
	- Не дублировать title, а дополнять его
	- Влиять на CTR в поисковой выдаче

100. Как работают структурированные данные?
	- Schema.org разметка помогает поисковикам понимать контент
	- Улучшает отображение в rich snippets (звездочки, цены, даты)
	- Форматы: JSON-LD (рекомендуется), Microdata, RDFa
	- Типы: Article, Product, Event, Organization, Review
	- Тестирование через Google Rich Results Test
	- Не гарантирует отображение в rich snippets

	- XML файл со списком всех страниц сайта для поисковиков
	- Содержит URL, дату последнего изменения, приоритет, частоту обновления
	- Помогает поисковикам находить и индексировать страницы

#### Современные возможности

103. Что такое Web Components?
	- Набор веб-стандартов для создания переиспользуемых HTML элементов
	- Состоит из: Custom Elements, Shadow DOM, HTML Templates
	- Позволяет создавать инкапсулированные компоненты без фреймворков
	- Работает во всех современных браузерах
	- Альтернатива React/Vue компонентам для простых случаев
	- Пример: создание собственного <my-button> элемента

104. Как работают Custom Elements?
	- Позволяют создавать собственные HTML элементы
	- Два типа: Autonomous (полностью новые) и Customized (расширение существующих)
	- Lifecycle callbacks: `connectedCallback`, `disconnectedCallback`, `attributeChangedCallback`
	- Регистрация через `customElements.define('my-element', MyElement)`
	- Пример: <my-button>Кнопка</my-button> с собственной логикой

105. Как работает Shadow DOM?
	- Инкапсулированный DOM внутри элемента, изолированный от основного документа
	- Стили и скрипты внутри Shadow DOM не влияют на внешний документ
	- Создается через `element.attachShadow({mode: 'open'})`
	- Решает проблему изоляции стилей в компонентах
	- Используется в Web Components для инкапсуляции

106. Как работают HTML Templates?
	- <template>: контейнер для HTML, который не рендерится до использования
	- Содержимое остается инертным до клонирования через JavaScript
	- Используется для создания переиспользуемых фрагментов HTML
	- Эффективнее создания элементов через `createElement`
	- Пример: <template id="user-card"><div class="card">...</div></template>

107. Как работает <template> элемент?
	- Хранит HTML разметку, которая не отображается на странице
	- Содержимое доступно через `document.getElementById('template-id').content`
	- Клонируется через `template.content.cloneNode(true)`
	- Используется для динамического создания контента
	- Поддерживается всеми современными браузерами

108. Как работает <slot> элемент?
	- Используется в Shadow DOM для создания "отверстий" для внешнего контента
	- Позволяет передавать контент в Web Component
	- Именованные слоты: <slot name="header">
	- Fallback контент: <slot>Контент по умолчанию</slot>
	- Пример: <my-card><h2 slot="header">Заголовок</h2></my-card>
